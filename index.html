¿Cómo Funcionan?

Las APIs funcionan mediante un modelo cliente-servidor. El cliente realiza una solicitud (Request) a través de HTTP, y el servidor responde (Response) con los datos solicitados en formato JSON.

Arquitectura Cliente-Servidor

Componentes:
  • Cliente: Envía solicitudes (navegador, app móvil)
  • Servidor: Procesa solicitudes y responde con datos
  • HTTP: Protocolo de comunicación
  • JSON: Formato de datos estándar

Desarrollo de Web API

¿Qué es una Web API?

Una Web API es un servicio web que permite la comunicación entre aplicaciones a través de Internet. Utiliza protocolos estándar como HTTP/HTTPS para intercambiar datos, generalmente en formato JSON.

Arquitectura REST

REST (Representational State Transfer) es un estilo arquitectónico que define cómo diseñar servicios web escalables y mantenibles.

Métodos HTTP

GET

Obtiene datos del servidor sin modificarlos.

GET /api/usuarios
POST

Crea nuevos recursos en el servidor.

POST /api/usuarios
PUT

Actualiza recursos existentes completamente.

PUT /api/usuarios/1
DELETE

Elimina recursos del servidor.

DELETE /api/usuarios/1

Ejemplo Práctico con Node.js + Express

const express = require('express'); const app = express(); app.use(express.json()); // Base de datos simulada let usuarios = [ { id: 1, nombre: 'Juan', email: 'juan@email.com' } ]; // GET - Obtener todos los usuarios app.get('/api/usuarios', (req, res) => { res.json(usuarios); }); // POST - Crear nuevo usuario app.post('/api/usuarios', (req, res) => { const nuevoUsuario = { id: usuarios.length + 1, nombre: req.body.nombre, email: req.body.email }; usuarios.push(nuevoUsuario); res.status(201).json(nuevoUsuario); }); // PUT - Actualizar usuario app.put('/api/usuarios/:id', (req, res) => { const usuario = usuarios.find(u => u.id == req.params.id); if (usuario) { usuario.nombre = req.body.nombre; usuario.email = req.body.email; res.json(usuario); } else { res.status(404).json({ error: 'Usuario no encontrado' }); } }); // DELETE - Eliminar usuario app.delete('/api/usuarios/:id', (req, res) => { usuarios = usuarios.filter(u => u.id != req.params.id); res.json({ mensaje: 'Usuario eliminado' }); }); app.listen(3000, () => console.log('Servidor en puerto 3000'));

Formato JSON

Arquitectura de Microservicios

Concepto de Microservicios

Los microservicios son un enfoque arquitectónico que divide una aplicación en servicios pequeños, independientes y desplegables que se comunican entre sí mediante APIs.

Monolito vs Microservicios

Ventajas
  • Escalabilidad Independiente: Cada servicio escala según su demanda
  • Independencia Tecnológica: Cada servicio puede usar diferentes tecnologías
  • Mantenibilidad: Código más limpio y fácil de mantener
  • Despliegue Independiente: Actualizar un servicio sin afectar otros
  • Resiliencia: El fallo de un servicio no derriba toda la aplicación

Ejemplo Práctico de Arquitectura

Una aplicación de e-commerce puede dividirse en microservicios como:

Servicio de Usuarios

Gestión de perfiles y autenticación

Servicio de Productos

Catálogo y gestión de inventario

Servicio de Pedidos

Gestión de compras y carrito

Servicio de Pagos

Manejo de Contenedores con Docker

¿Por qué Docker?
Docker resuelve el problema "funciona en mi máquina" permitiendo encapsular aplicaciones con todas sus dependencias en contenedores que funcionan igual en cualquier ambiente.

Conceptos Clave

Contenedor

Instancia ejecutable de una imagen Docker. Incluye todo: código, runtime, dependencias.

Imagen

Plantilla de solo lectura que define todo lo necesario para crear un contenedor.

Dockerfile

Archivo de texto que define los pasos para construir una imagen Docker.

Tutorial: Contenedores Docker

Video explicativo sobre cómo funcionan los contenedores Docker

Desarrollo de Aplicaciones para la Nube

¿Qué es Cloud Computing?

Cloud Computing es el acceso bajo demanda a recursos computacionales (servidores, almacenamiento, bases de datos) a través de Internet. No requiere infraestructura física propia.

Características Principales:
  • Acceso bajo demanda
  • Escalabilidad elástica
  • Pago por uso
  • Disponibilidad global
  • Seguridad robusta
  • Mantenimiento automático

Modelos de Servicio en la Nube

IaaS
Infrastructure as a Service

Acceso a infraestructura virtual (servidores, almacenamiento). Ejemplo: AWS EC2, Azure VMs

Control: 40%
PaaS
Platform as a Service

Plataforma completa para desarrollar y desplegar aplicaciones. Ejemplo: Heroku, Google App Engine

Control: 70%
SaaS
Software as a Service

Aplicaciones listas para usar en la nube. Ejemplo: Office 365, Salesforce, Slack

Control: 100%

Despliegue de Web API en la Nube

  1. Crear cuenta en AWS
  2. Usar servicio EC2 (máquina virtual) o ECS (contenedores)
  3. Configurar security groups y subnets
  4. Desplegar aplicación con Docker o directamente

  1. Crear suscripción en Azure
  2. Usar App Service para aplicaciones Node.js/ASP.NET
  3. O Azure Container Instances para Docker
  4. Usar Azure SQL Database o MySQL flexible
  5. Configurar Application Gateway
  6. Implementar CI/CD con Azure DevOps

Pasos para desplegar en Google Cloud:
  1. Crear proyecto en Google Cloud Console
  2. Usar Cloud Run para aplicaciones serverless
  3. O GKE (Google Kubernetes Engine) para orquestación
  4. Usar Cloud SQL para bases de datos
  5. Implementar Cloud Load Balancing
  6. Configurar Cloud Build para CI/CD

¿Qué es una API?
Definición Técnica

Una API (Application Programming Interface) es un contrato entre dos aplicaciones que define cómo pueden comunicarse entre sí, intercambiando solicitudes y respuestas.

Analogía del Mundo Real

Imagina un restaurante: el camarero es la API. El cliente (aplicación) hace un pedido (request), el camarero lo lleva a la cocina (servidor), y regresa con la comida (response).

Componentes Principales
  • Endpoint: URL del servicio (ejemplo: /api/usuarios)
  • Método HTTP: GET, POST, PUT, DELETE
  • Headers: Metadatos de la solicitud
  • Body: Datos enviados (generalmente en JSON)
  • Status Code: Código de respuesta (200, 404, 500, etc.)
Ejemplo Real
// Solicitud GET GET /api/usuarios/1 HTTP/1.1 Host: api.ejemplo.com Authorization: Bearer token123 // Respuesta 200 OK { "id": 1, "nombre": "Juan", "email": "juan@email.com" }
